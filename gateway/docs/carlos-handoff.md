# Gateway Integration Handoff (For Carlos)

Hey Carlos! The Gateway backend is fully built and tested. The Agent-to-UI contract is officially enforced, and the backend is ready for you to hook up your React frontend for the midweek integration.

## 🚀 How to Run the Backend Locally

To test your UI against the API, you need to run the Gateway server in the background:

1. Open a new terminal and navigate to the new gateway directory:
   ```bash
   cd greenlight-new/gateway
   ```
2. Install dependencies (first time only):
   ```bash
   npm install
   ```
3. Start the HTTP API:
   ```bash
   npm run dev:api
   ```
   *The server will start on `http://localhost:3000`. (CORS is already configured to accept requests from your React app running on `http://localhost:5173`).*

## 🔗 What You Need to Build

The Vayu agent generates a review link that looks like this:
`http://localhost:5173/review/<SESSION_ID>`

You need to build the `/review/:session_id` route in your React app. When a human opens that page, your UI should interact with the Gateway API using the two endpoints below.

### 1. Fetch the Design (`GET`)
When the page loads, fetch the session state and the A2UI design object generated by the AI:

**Request:**
```http
GET http://localhost:3000/session/<SESSION_ID>
```

**Response:**
```json
{
  "session_id": "d8d34abf-1d1e-41de-93fe-f568bd2a5044",
  "state": "in_review",
  "design": {
    "schema_version": "0.1.1",
    "canvas": { "preset": "square", "width": 1080, "height": 1080 },
    "layers": { ... },
    "meta": { ... }
  },
  "result": null
}
```
*(Note: Hitting this endpoint automatically transitions the session state from `posted` to `in_review` so the AI knows the human is looking at it).*

### 2. Submit the Human's Decision (`POST`)
When the user clicks "Download" or "Send Back", post their final result back to the Gateway.

**Request:**
```http
POST http://localhost:3000/session/<SESSION_ID>/result
Content-Type: application/json
```

**Payload Body:**
```json
{
  "state": "sent_back", // MUST be either "approved_downloaded" OR "sent_back"
  "human_note": "Can you change the background to blue?", // Optional context for the AI
  "final_design": {
    // The exact A2UI design object, including any tweaks the human made
    "schema_version": "0.1.1",
    "canvas": { ... },
    "layers": { ... },
    "meta": { ... }
  }
}
```

**Response:**
```json
{
  "session_id": "d8d34abf-1d1e-41de-93fe-f568bd2a5044",
  "state": "sent_back"
}
```

## 🧪 Testing Tip
To make testing easier without polluting Git, `gateway/sessions.json` is gitignored so local modifications don't cause merge conflicts. 

Instead, the database automatically **auto-seeds itself**! The very first time you start the Express server, it detects if the database file is missing, generates `gateway/sessions.json`, and pre-seeds it with a dummy review session (`test-1234`) in the `"posted"` state.

You can immediately test your React UI by navigating to `http://localhost:5173/review/test-1234` without needing the AI agent to generate anything.
